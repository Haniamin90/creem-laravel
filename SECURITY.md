# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability within the CREEM Laravel package, please report it responsibly.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email security concerns to: **support@creem.io**

Include:
- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix release**: As soon as possible, depending on severity

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | Yes       |

## Scope

This policy covers the `creem/laravel` Composer package. For vulnerabilities in the CREEM API itself, please contact CREEM directly at [creem.io](https://creem.io).
